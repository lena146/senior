# 시니어 여행 가이드

시니어의 **건강 상태**와 **성향**을 반영한 여행 계획 및 여행지 AI 가이드 웹 서비스입니다.  
모바일에서 사용하기 좋도록 반응형으로 구성되어 있습니다.

## 기능

- **건강·성향 설문**: 이동 편의성, 휴식 필요도, 식이 제한, 의료·약 복용, 여행 스타일, 기간, 원하는 지역 입력
- **맞춤 여행 계획**: 설문 결과에 맞는 국내 여행지 추천 및 건강 맞춤 팁
- **여행지 AI 가이드**: 선택한 여행지별로 건강·성향을 반영한 추천 코스, 이동 팁, 주의사항

## 실행 방법

별도 빌드 없이 브라우저에서 바로 열 수 있습니다.

```bash
# 프로젝트 폴더에서 로컬 서버 실행 (선택)
npx serve .
# 또는
python3 -m http.server 8080
```

또는 `index.html` 파일을 브라우저에서 직접 열어도 됩니다.

- **로컬 서버 사용 시**: `http://localhost:3000` (serve) 또는 `http://localhost:8080` (python) 접속
- **파일 직접 열기**: `index.html` 더블클릭 또는 브라우저에서 파일 열기

## 프로젝트 구조

```
vibe/
├── index.html      # 메인 페이지 (설문 + 계획 + 가이드)
├── css/
│   └── style.css   # 모바일 최적화·시니어 친화 스타일
├── js/
│   └── app.js      # 설문 처리, 추천 로직, 가이드 생성
└── README.md
```

## 모바일 최적화

- 뷰포트 및 터치 최소 크기(48px) 적용
- 모바일 우선 반응형 CSS (600px 이상에서 여백·버튼 조정)
- `safe-area-inset`으로 노치·홈 인디케이터 대응
- `prefers-reduced-motion` 지원

## 시니어 친화 UI

- 본문 17px, 제목 22~26px 등 가독성 위주 폰트 크기
- 높은 대비의 텍스트/배경
- 큰 터치 영역과 단순한 네비게이션
- Noto Sans KR 폰트 로드

## 확장 (실제 AI 연동)

현재 가이드는 설문 데이터와 여행지별 사전 데이터를 조합해 생성됩니다.  
실제 LLM API(OpenAI, Claude 등)를 붙이려면:

1. `js/app.js`의 `generateGuideHTML()` 호출 전에 API 요청을 보내고
2. 응답을 파싱해 `guide-content`에 넣는 방식으로 교체하면 됩니다.
3. API 키는 반드시 서버 측에서만 사용하거나, 사용자 입력으로 받아 클라이언트에서 호출할 때만 사용하세요.
